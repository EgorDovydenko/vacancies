# 🤖 Vacancies Bot — Telegram-бот для публикации IT-вакансий

Автоматический агрегатор вакансий для **frontend**, **backend** и **fullstack** разработчиков из **Беларуси, России, Грузии, Армении и Казахстана**. Публикует отформатированные карточки вакансий в Telegram-канал по расписанию.

## ⚙️ Как это работает

```
┌─────────────────────────────────────────────────┐
│                   Планировщик (Cron)             │
│          запускается каждые N минут              │
└─────────────────┬───────────────────────────────┘
                  │
        ┌─────────▼──────────┐
        │   Парсеры / API    │
        │  hh.ru  rabota.by  │
        │      dev.by        │
        └─────────┬──────────┘
                  │  список новых вакансий
        ┌─────────▼──────────┐
        │  Фильтр дублей     │
        │  (data/published   │
        │     .json)         │
        └─────────┬──────────┘
                  │  только новые
        ┌─────────▼──────────┐
        │   Форматирование   │
        │   карточки поста   │
        └─────────┬──────────┘
                  │
        ┌─────────▼──────────┐
        │  Telegram Bot API  │
        │  → пост в канал    │
        └────────────────────┘
```

1. По расписанию (cron) запускается цикл публикации
2. Каждый источник опрашивается для каждой поддерживаемой страны
3. Вакансии фильтруются по уже опубликованным (хранятся в `data/published.json`)
4. Новые вакансии форматируются в единый стиль и отправляются в канал
5. ID опубликованных вакансий сохраняются, чтобы не дублировать посты

## 🌐 Источники вакансий

| Источник               | Страны                            | Метод                | API-ключ        |
| ---------------------- | --------------------------------- | -------------------- | --------------- |
| **hh.ru** (HeadHunter) | 🇷🇺 RU, 🇧🇾 BY, 🇰🇿 KZ, 🇦🇲 AM, 🇬🇪 GE | Официальный REST API | Не обязателен\* |
| **rabota.by**          | 🇧🇾 BY                             | HTML-парсинг         | Не нужен        |
| **dev.by**             | 🇧🇾 BY                             | HTML-парсинг         | Не нужен        |

> \* HH.ru Public API работает без токена с лимитом ~3 запросов/сек и 2000 вакансий/день.
> Токен нужен только если превысишь лимиты (регистрация бесплатна на [dev.hh.ru](https://dev.hh.ru)).

### Почему именно эти источники?

- **hh.ru** — крупнейшая площадка СНГ, единый API для всех 5 стран через параметр `area`
- **rabota.by** — №1 по вакансиям в Беларуси
- **dev.by** — IT-специализированная платформа Беларуси

## 🔑 Настройка .env

Скопируй `.env.example` в `.env` и заполни:

```env
TELEGRAM_BOT_TOKEN=1234567890:AAF...   # Токен бота от @BotFather
TELEGRAM_CHANNEL_ID=@mychannel         # Username канала или числовой ID

HH_API_TOKEN=                          # Опционально (см. ниже)

PUBLISH_INTERVAL_MINUTES=60            # Как часто запускать парсинг
MAX_VACANCIES_PER_RUN=5                # Макс. вакансий за раз на источник/страну
REQUEST_DELAY_MS=2000                  # Задержка между запросами (мс)
LOG_LEVEL=info                         # error | warn | info | debug
```

## 🗝 Получение API-ключей

### 🤖 Telegram Bot Token (ОБЯЗАТЕЛЬНО)

1. Открой Telegram, найди **@BotFather**
2. Напиши `/newbot`
3. Задай имя бота (например: `Dev Vacancies RU Bot`)
4. Задай username (например: `dev_vacancies_ru_bot`)
5. BotFather отдаст токен вида `1234567890:AAFxxxxx` — это и есть `TELEGRAM_BOT_TOKEN`

### 📢 ID Telegram-канала (ОБЯЗАТЕЛЬНО)

**Вариант А — username канала:**

```env
TELEGRAM_CHANNEL_ID=@your_channel_username
```

**Вариант Б — числовой ID (для приватных каналов):**

1. Добавь **@userinfobot** в канал как администратора
2. Напиши ему `/start` в личку — он покажет ID
3. Числовой ID обычно начинается на `-100...`

```env
TELEGRAM_CHANNEL_ID=-1001234567890
```

> ⚠️ **Важно:** Добавь своего бота в канал как **администратора** с правом публикации сообщений!

### 🟡 HH.ru API Token (ОПЦИОНАЛЬНО)

Нужен только если планируешь много запросов (>2000 вакансий/день):

1. Зарегистрируйся на [dev.hh.ru](https://dev.hh.ru)
2. Создай приложение → получи `client_id` и `client_secret`
3. Для токена типа "приложение" (без авторизации пользователя) используй Client Credentials flow
4. Вставь токен в `HH_API_TOKEN`

## 💰 Как монетизировать канал

1. **Telegram Ads (официальная реклама)** — доступна при достижении **1000 подписчиков**. Настраивается в [Fragment](https://fragment.com) или через рекламный кабинет Telegram.

2. **Прямая реклама** — компании сами пишут с предложением разместить вакансию/баннер. Типичный прайс: $20–100 за пост при аудитории 1k–10k.

3. **Платные размещения вакансий** — HR-ы и компании платят за гарантированный пост. Можно оформить через боти-форму или вручную.

4. **Партнёрства** — с IT-курсами, буткемпами, сервисами резюме (Resume.io, Хантфлоу и т.д.)

5. **Закреплённый спонсорский пост** — баннер в закрепе канала, обновляется раз в неделю.

### 📈 Стратегия роста

- Публикуй **10–20 вакансий в день** — оптимальный ритм без спама
- Добавь **рубрики**: `#relocation`, `#senior`, `#junior`, `#стажировка`
- Кросс-постись в тематические чаты Telegram
- Упоминай канал на LinkedIn, в IT-чатах Discord/Slack

## ➕ Добавление новых источников

1. Создай файл `src/sources/mysource.ts`:

```typescript
import { Source, ParsedVacancy, Country } from "../types";

export const mySource: Source = {
  name: "mysource.com",
  countries: ["RU", "BY"],

  async scrape(country: Country): Promise<ParsedVacancy[]> {
    // Твоя логика парсинга
    return [];
  },
};
```

2. Добавь его в реестр `src/sources/index.ts`:

```typescript
import { mySource } from "./mysource";

export const sources: Source[] = [
  hhSource,
  rabotaBySource,
  devBySource,
  mySource, // ← добавить сюда
];
```

Готово — источник автоматически подхватится планировщиком.
